myFlix API Documentation

This is a backend "movie" application build using Node.js. This application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create list of their favorite movies. User authentication and authorization is implementated using JWT tokens.

Sl No Request Endpoint Url Method type Request data format Response data format
1 Return a list of all movies /movies GET - A JSON array holding data about movies
2 Return data about a single movie by title /movies/:Title GET - A JSON object holding data about a single movie, containing the title, description, genre, director, image URL, whether it's featured or not.
3 Return data about a genre by name/title /movies/genre/:Name GET - A JSON object holding movie data which matches genre name.
4 Return data about a director by name /directors/:Name GET - A JSON object holding the name, bio, birthdate and death year of a director.
5 Allow new users to register /users POST A JSON Object holding data about the new user.

Example:
{
"Username": "user1",
"Password": "password1",
"Email": "user1@gmail.com"
"Birthday": "09/02/2000"
}
A JSON Object holding data about the new user created, with an id.

Example:
{
"_id": XXXXX,
"Username": "user1",
"Password": "password1",
"Email": "user1@gmail.com"
"Birthday": "09/02/2000"
}
6 Allow users to update their user info /users/:Username PUT A JSON Object holding data the user information along with field to update. A JSON Object holding the user's updated data.
7 Allow users to add a movie to their list of favorites /users/:UserID/movies/:MovieID POST - A JSON Object holding the user's updated favorite movie list.
8 Allow users to remove a movie from their list of favorites /users/:UserID/movies/:MovieID DELETE - A JSON Object holding the user's altered favorite movie list.
9 Allow existing user to deregister /users/:Username DELETE - A text message indicating whether the user was successfully deleted.
